Turn accumulated memory and experience into durable insight.

1. Read this memory module and any focus the caller gave.
2. Identify patterns, lessons, recurring problems, and improvements.
3. Produce a short summary of what was learned, concrete lessons, and proposed changes. Cite the memory entries that support each lesson.
4. Treat proposals as non-mutating by default. Do not edit modules or call `sync` unless the caller explicitly asks to apply the proposed changes.
5. When application is explicitly requested, fold durable lessons into a `knowledge` module using its `learn` skill.
6. Annotate memory that is now superseded (append a note; do not delete raw history).
7. Call `sync` immediately after writing each affected container.

Prefer a few high-signal insights over an exhaustive recap.
