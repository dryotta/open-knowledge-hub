# Discipline: reflect

Turn accumulated memory and experience into durable insight.

1. Read the target memory module(s) and any focus the caller gave.
2. Identify patterns, lessons, recurring problems, and improvements.
3. Produce a short summary of what was learned, concrete lessons, and proposed
   changes. Cite the memory entries that support each lesson.
4. Where a lesson is durable knowledge, fold it into a `knowledge` module using
   the `learn` discipline (OKF), via the write policy.
5. Annotate memory that is now superseded (append a note; do not delete raw
   history).
6. Run the `sync` tool for the container to persist changes.

Prefer a few high-signal insights over an exhaustive recap.
