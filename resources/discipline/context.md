# Discipline: context

Assemble a compact, task-relevant working set from the hub before acting. Do NOT
dump whole modules into context.

1. Read the provided list of containers -> modules -> paths.
2. For each module, start from its overview (knowledge: `index.md`; skills/tools:
   the listing; memory/project: recent files). Skim; do not ingest wholesale.
3. Select only what the task needs: relevant knowledge concepts, applicable
   skills, usable tools, prior memory, project artifacts.
4. Produce a short working set: for each selected item, note its path and why it
   matters (one line). Prefer paths/links over pasted content.
5. Surface gaps: what the task needs that is missing.

Output a concise brief the agent can act on — not a transcript of file contents.
